## Architecture

This is a **Next.js 15 App Router** project using TypeScript, Tailwind CSS, and React 19.

### Multi-section landing page

The site has four distinct sections, each with its own route, layout, and component tree:

| Route | Purpose |
|---|---|
| `/business` | Main landing page (default â€” root `/` redirects here) |
| `/candidate` | Job marketplace / candidate portal |
| `/marketing` | Content360 marketing product |
| `/legacy` | Legacy page (not actively maintained) |

Each section has a `layout.tsx` for SEO metadata and may have its own CSS file (e.g., `app/business/business.css`).

### Component organization

- `app/<section>/` â€” Page components, section-specific components, hooks, and utilities
- `src/common/` â€” Shared components, hooks, translations, and i18n config
- `src/candidate/` â€” Candidate-specific components (57 files)
- `src/legacy/` â€” Legacy components including Gemini API utilities
- `src/contexts/` â€” React Context providers (LanguageContext)

### Internationalization

Custom i18n system layered on top of i18next:

- Languages: `en` and `es`
- Detection priority: URL querystring `?lg=` â†’ localStorage `tlang` â†’ browser navigator
- Translation strings live in `src/common/translations/en.ts` and `es.ts` as nested objects
- Language state is managed via `src/contexts/LanguageContext.tsx`
- Each section has a `useTranslation.ts` hook (e.g., `app/business/useTranslation.ts`) that selects the right subtree of translations
- `src/common/components/I18nProvider.tsx` wraps the app and handles detection/persistence

### AI demo endpoints

`app/api/demo/` contains serverless routes (business-plan, market-research, financial-projections, etc.) that all call Google Gemini via `src/legacy/utils/geminiApi`. These require `NEXT_PUBLIC_GOOGLE_API_KEY_1` and `NEXT_PUBLIC_GOOGLE_API_KEY_2` env vars (see `.env.example`).

### Styling

- Tailwind CSS with custom brand colors (`moil-navy`, `moil-blue`, `moil-orange`, `moil-green`) defined in `tailwind.config.js`
- Custom breakpoints: `vsm` (320px), `xsm` (400px), `zsm` (480px), `tablet`, `ipad`, `desktop`, `xtraxl` (1440px), `xxlarge` (3200px)
- Global styles split across `app/globals.css` (2009 lines) and `styles/globals.css`
- CSS custom properties for theme variables; dark mode infrastructure exists in `useBusinessUi.ts` but is not fully implemented

### Path aliases

`~~/*` and `@/*` both resolve to the repo root (configured in `tsconfig.json`).

### Deployment

Deployed on Vercel. `vercel.json` sets install command to `yarn install`. TypeScript and ESLint build errors can be bypassed via `NEXT_PUBLIC_IGNORE_BUILD_ERROR=true` env var (see `next.config.js`).

## SEO Tooling (Toprank)

This repo enables the [Toprank](https://github.com/nowork-studio/toprank) Claude Code plugin (AdsAgent's Google Ads + SEO suite) via `.claude/settings.json`. Anyone running Claude Code in this repo gets the `/toprank:*` skill namespace.

### Available skills

| Command | Use |
|---|---|
| `/toprank:seo-analysis` | Full SEO audit using Google Search Console data (90-day window). Quick wins, traffic drops, technical issues, 30-day action plan. |
| `/toprank:content-writer` | E-E-A-T blog posts and landing page content. Includes JSON-LD, internal linking plan, metadata. |
| `/toprank:keyword-research` | Seed-to-cluster keyword discovery with intent classification and a prioritized content calendar. |
| `/toprank:meta-tags-optimizer` | Title, description, OG/Twitter card variants with CTR estimates. |
| `/toprank:schema-markup-generator` | JSON-LD for FAQ, HowTo, Article, Product, LocalBusiness, etc. |
| `/toprank:setup-cms` | One-time CMS connector setup (WordPress, Strapi, Contentful, Ghost). |

`/toprank:ads`, `/toprank:ads-audit`, `/toprank:ads-copy`, `/toprank:ads-landing` are also available for Google Ads management; they require an AdsAgent API key (see `ADSAGENT_API_KEY` in [adsagent.org](https://www.adsagent.org)).

### Site context

- Canonical URL: `https://www.moilapp.com` (from `src/common/constants/baseUrl.tsx`).
- Sitemap: generated by `app/sitemap.ts`, served at `/sitemap.xml`.
- Robots: generated by `app/robots.ts` (already blocks AhrefsBot, SemrushBot, etc.).
- Existing per-section metadata + JSON-LD lives in `app/layout.tsx` and each `app/<section>/layout.tsx`. Toprank should treat these as ground truth, not duplicate them.
- No blog directory exists yet. `/toprank:content-writer` output should land as a draft (e.g. `docs/seo-drafts/`) until `app/blog/` (or equivalent) is scaffolded in a future task.

### GSC authentication (one-time, local only)

The first invocation of `/toprank:seo-analysis` runs `preflight.py`, which auto-detects gcloud, runs `gcloud auth application-default login` if needed, and enables the Search Console API. Requirements:

- `gcloud` CLI installed (`brew install google-cloud-sdk` on Mac).
- The Google account used must have at least Restricted user access on the GSC property for `moilapp.com`.
- Web Claude Code sessions cannot complete the OAuth flow. Run Toprank from the local Claude Code CLI.

### Pre-flight before the first `/toprank:seo-analysis`

- Verify `moilapp.com` is a verified property in Search Console (Domain property `sc-domain:moilapp.com` preferred over URL-prefix).
- Confirm the sitemap `https://www.moilapp.com/sitemap.xml` is submitted in Search Console.
- The placeholder verification meta tags at `app/layout.tsx:99-103` are commented out and orthogonal â€” Toprank uses API auth, not the meta tag â€” but if the property was never verified by any method, the audit will return empty data.
